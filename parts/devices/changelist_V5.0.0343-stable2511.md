**固件/系统接口更新**

- 彻底摆脱了旧世界 (ABI1.0) 环境 (V3.1) 的向前兼容包袱，全平台默认均切换为 V4.0 系统接口规范（[参考链接](https://github.com/tianocore/edk2/)）
- 将工具链链从 GCC 8.3 切到 GCC 12/13/14，完全复用 EDK2 社区工具链
- 设置界面去除旧世界 (Legacy) 启动方式选项，转向 VA=>PA 1:1 地址映射内存管理模式
- 告别老旧的 UDK2018，全面切换到 EDK2025，龙架构大部分基础设施 API 均复用 EDK2 上游实现

**模块升级**

- 新增 3B6000、3C6000/{S,D,Q} 及 2K3000/3B6000M 等系列单路/多路芯片支持
- 升级 PHY、MRC、SMC、IPMI、MultiArch、eMMC 等软硬件代码模块

**新增功能**

通用特性：

- 引入 MultiArchUefiPkgs，支持第三方外设卡的 x86 Option ROM/EFI 模拟及过滤（可配置是否开启）
- 新增内存频率调整选项，可在必要时降低内存频率提高兼容性（不同机型可配频率可能会有区别；界面提供恢复选项，亦可拔除 CMOS 电池复位设置）
- 加入可调界面及控制台分辨率选项
- 新增 DSM#5 选项，允许控制 Linux 内核 PCIe 资源分配策略
- 新增华瑞 RAID 控制器、网讯 25/40GbE 网卡等国产外设驱动支持
- 新增多显卡配置，可允许多显卡在 UEFI 下同时显示
- 新增 TPM 及安全启动功能：支持 SHA-384、SHA-512 及 SM3 等算法（TPM 需硬件支持）
- 分离 PCIe、硬盘 及 USB 信息界面，加入硬盘详细信息展示界面
- 更新 SRAT 版本到 6.5，支持 x2APIC 扩展
- 同步 EDK2 上游的龙架构多核唤醒、异常、MMU 等逻辑

硬件平台专属特性：

- 新增面向 3B6000 及 3C6000/S/D/Q 系列新增基于龙链互联 (LCL) 的 PCIe 4.0 设备训练支持
- 新增 2K3000 eMMC 支持
- 新增 3C6000 的 ACPI Thermal Zone/HWMon 监测支持
- 新增服务器平台的 ACPI EDAC 支持，Linux 内核可查询 ECC 状态
- 完善部分机型基于系统管理控制器 (System Management Controller, SMC) 的功耗限制、睿频及风扇监测控制等功能
- 支持部分机型通过安全引擎 (Security Engine, SE) 设备模式上报 DSDT 表内资源描述

**功能修复与优化**

通用问题修复：

- 修复单路服务器平台外接显卡和 BMC 显卡同时在位情况下，优先显示选择 All Video 可能触发宕机的问题
- 修复开机屏幕固件版本及编译时间的显示乱码
- 优化 SMBIOS 表支持，主要修复 DMI Type 7 逻辑问题
- 修复写入 RTC LP 时发生意外停止的问题
- 修复部分 ACPI 表问题
- 修复 SATA 口控制器开关顺序错乱的问题
- 修复个别中文翻译问题
- 修复启动时徽标可能偶尔闪黑的问题

平台专属修复和调整：

- 修复部分机型未正常显示 GMAC 网络设备信息的问题
- 调整服务器平台 7A2000 桥片频率为 750MHz
- 优化 AST2500 和 LS2K0500 BMC 命令协议兼容性，修复 IPMI 显示问题和个别命令出错的问题
- 修复 2K0500 BMC 固件版本高于 v2.2.2 时，BMC 网页界面不同步 SMBIOS 信息的问题
- 修复部分平台 SATA P-N 反接及 PCIe LnkCap2 设置错误的问题
- 修复 2K0500 BMC 局域网串口 (Serial-over-LAN, SOL) 无输出的问题
- 修复多款芯片的 PPTT 表
- 关闭 PTW 功能，避免部分内核页表问题导致数据出错，待 Linux 内核普遍带有修复后重新打开该特性
- 移除部分平台的 ACPI GPIO 资源描述，避免系统兼容性问题，待系统普遍补齐支持后重新补充

功能优化：

- 优化部分固件界面选项
- 优化 BMC 在位动态检测与设置界面显示，新增 FRU 数据扩展、USB 网口界面控制等
